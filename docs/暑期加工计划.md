# 纯python仿真器开发任务分配

感知：LiDAR 模拟器，赛道加载模拟器

定位建图：INS 模拟器

规控：车辆运动学模型 ，CAN 模拟器

## 阶段一：各组单独完成仿真器的设计与验收（7\.7\.13）

每组写一个文档解释仿真器的设计原理，以及自己组模块的整个work flow\(言简意赅，1\-2页）



## 阶段二：各组merge代码，完成全链路仿真（7\.14\-7\.20）

各组负责人对接，merge代码，各组记得统一坐标系。

建图组：将map\_saver建的图与感知的赛道yaml文件对比，重点查看锥桶位置偏移率和锥桶数量信息是否正确，确认建图流程无误。

感知组：确保地面分割正常工作  ，分析锥桶检出率，避免漏检率过高

规划控制组：

1\.验证生成轨迹合理性， 轨迹沿赛道中心线，不蛇行、不切弯、不冲出赛道 

2\.横向偏差  逐帧计算车辆到中心线的距离 均值 。\< 0\.3m，最大值 \< 1\.0m（弯道可放宽）

3\.弯道通过质量  单独统计弯道路段的横向偏差 。 \< 1\.0m，过弯后 5m 内收敛回中心线

并给出生成轨迹可视化图片

最终评估指标:

|1\.端到端延迟|从 /hesai/pandar 发布时间戳到 /control/command    发布时间戳 |总耗时\<100ms|
|---|---|---|
|2\.可复现性|相同 seed 跑两次，对比 trajectory\.csv|轨迹完全一致（逐帧逐值相等）|
||||

完成后，写一份整个仿真器的可行性分析。
